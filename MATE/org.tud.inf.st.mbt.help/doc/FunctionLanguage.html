Function Language

Functions allow to compute information about the model's state at simulation time. Thus, it is not necessary to define each derivable piece of information explicitely. MATE supports the following data types for functions:

  • arithmetrical (float)
  • logic (bool)
  • data (data values from pre-defined finite domains)
  • node (pre-defined variables and data structure nodes)

Arithmetrical Operators

Supported data types:

  • int, e.g., 1
  • long, e.g., 123457890
  • float, e.g., 1.23

The up-to-dat supported operators are:

  • multiplication '*'
  • division '/'
  • addition '+' and '-'
  • brackets '(' ')'

Additionally, several functions on bags can be queried:

  • size(<id.of.bag>)
  • next(<id.of.bag>), the next free element
  • free(<id.of.bag>), the first free element
  • index(<id.of.bag>,<needle>), the index of data element needle in a bag
  • count(<id.of.bag>,<needle>), the number of occurances of needle in the bag

A valid arithmetrical expression is for instance: 

(1*2)+(3-4)/5

Logic Operators

Most of logic operators are defined in Rules Language. Additionally, arithmetrical functions can be compared with boolean results. Comparison operators are:

  • >,<,=
  • the 
    "<containedValue> in <container>" 
    operator checks if a data value is contained by another equivalence data class

Data Operators

  • ?<NodeFunction>, cf. next section. Queries the value of a specific pre-defined data node.

Node Operators

  • $<id.to.a.node>
  • $<id.of.bag>[idx], a specific element in a bag

Common Operators for All Function Types

Common operators are type-safe!

  • References <id.of.function> to functions defined in a Defintions file.
  • Cases (not for logic functions): 
              | <Logic Function> = <Function>
    Example: 
    | ?a=2 = 4 | ?a=3 = 5
