Rules Language

Rules are propositions on the simulation model's state. The computational result of a rule is boolean. The following types of atoms are supported:

Class Syntax Examples 
True
 
otherwise | true
 
true
 
FeatureAtom
 
<id.of.feature> | <id.of.feature>[<id.of.version>]
 
Motor[Diesel]
 
FunctionAtom
 
[<a logic function>]
 
[1  > 2]

These atoms can be composed by serveral operators:

  • not
  • and
  • or
  • implication '->'
  • equivalence '<->'
  • brackets '(' ')'

A valid rule is for instance:

(Motor[Diesel] and ?a>2) -> Wheels
