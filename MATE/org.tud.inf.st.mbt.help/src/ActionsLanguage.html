Actions Language

With the actions language, several types of test actions can be defined:
  1. Term Actions: Arbitrary messages for communicating with system under test. The syntax of term actions is defined as Term Language.
  2. Data Exchange: Upload synchronizable data and state entities with the system under test.
  3. Reconfiguration Actions: Change data values and feature states of the variability model. Only these actions can be used in Operational Configuration Models (OCMs).
  4. Event Actions: Delegate control flow by producing events and logic time.
  5. Reference: Call action from a definitions file: <id.of.action>(<ListOfParameters>)
  6. Fail action:A special action 
    &fail
    allows to force a failed test outcome.
All actions can be sequenced by separating them by a ";"; terminal. For instance: 
action1;action2;action3;

Data Exchange Actions

Class Syntax Example 
GetDataAction
 
&get value <id.of.node>
 
&get value x
 
GetFeatureStateAction
 
&get state <id.of.feature>
 
&get state f

Reconfiguration Actions

Class Syntax Example 
SetDataAction
 
<NodeFunction> = <DataFunction>
 
x = v0
 
RemoveBagAction
 
&unset <id.of.bag>[idx]
 
&unset b[0]
 
ActivateFeatureAction
 
&activate <id.of.feature>
 
&activate Motor
 
DeactivateFeatureAction
 
&deactivate <id.of.feature>
 
&deactivate Motor

Class Syntax Example
